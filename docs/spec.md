# ヤギの鼻の中アクションゲーム 仕様書

> このファイルは `description.md`（ユーザのメモ）を作業AI向けに構造化したもの。
> 仕様の一次ソースはあくまで `description.md`。矛盾があれば description.md とユーザの指示を優先する。

## 1. 全体像

- ジャンル: ブラウザで動く 2D 物理アクションゲーム（オフライン専用）
- 対応環境: PC（キーボード）とスマホ（タッチ）の両対応
- 技術スタック: TypeScript + Vite + PixiJS（描画）+ Matter.js（物理）
- 演出方針: キャラクターや背景は Live2D のようにボーンベースでなめらかに変形・動作する
- アート: 画像素材はヤギ（referenceAssets/images）のみ。それ以外（主人公・ぬいぐるみ・鼻内部等）はコードで描画する
- 数値パラメータは `src/config/params.ts` に集約する（ユーザが直接調整する。**AIは値を勝手に元へ戻さない**）

## 2. 登場キャラクター

### 主人公（プレイヤー操作）
- 小型の人型。とても小さく、ヤギの鼻の穴の直径の半分より少し小さいサイズ
- 状態: 「直立」と「うつ伏せ」の2状態。開始時は直立

### 巨大ヤギ（NPC）
- 見た目は referenceAssets/images のヤギ。絶対サイズは普通のヤギだが、主人公から見ると巨大
- 小さな生き物を見つけると匂いを嗅ごうとして、吸引力で誤って鼻に吸い込む習性
- 鼻の中を刺激される（壁への接触・鼻毛が抜ける）と「くすぐりメーター」がたまり、閾値超過でくしゃみをする

## 3. シーンフロー

```
タイトル → プロローグ → ゲーム本編 → クリア演出 or ゲームオーバー演出 → UI（リトライ / タイトルへ）
```

## 4. プロローグ演出

1. 舞台は牧場のヤギ舎。主人公は干し草の上でぬいぐるみを抱いてくつろいでいる。ヤギは辺りを歩いている
2. ヤギが主人公を見つけ、不思議がって主人公のほうへ歩いてくる
3. **カメラを主人公目線に切り替え**。ヤギが顔を主人公の真ん前まで近づける
   - この時、画面は巨大なヤギの鼻のドアップ。ヤギは主人公に対して正面向き
4. ヤギが鼻から巨大な吸い込みを開始
   - カメラを小刻みに振動させつつ、主人公の見かけサイズが変わらないようにズームして追従
   - 鼻の動きはボーンベースでなめらかに動かす
5. 主人公（とぬいぐるみ）が鼻の穴に吸い込まれ、最終的に画面全体が真っ暗になる
6. 鼻の中で目覚める。ぬいぐるみが無いことに気づく（鼻の奥に詰まっている）
   - 導入テキスト: 鼻の壁を刺激するとくしゃみで外に弾き飛ばされる。刺激しないように鼻の奥まで進む

## 5. ゲーム本編

### 5.1 舞台・地形
- 舞台はヤギの鼻の中。背景には goatRight.png の顔部分を拡大したものを使い、これもボーンベースで顔や口がなめらかに動く
- 洞窟（鼻腔）は円形チューブ状:
  - 鼻の穴の入り口 〜 鼻の奥（ヤギの耳の位置あたり）まで伸びる
  - 半径: 平均は主人公のサイズよりわずかに大きい程度。場所によって少し大きい/小さい変動あり
  - 経路は上下方向に少しグネグネ蛇行する
- 鼻の内壁は少し弾力のある物理挙動
- 鼻毛: 地面または天井から、ある程度等間隔に、主人公と同程度の長さで生える。ばねのようになびく物理挙動
- ヤギは呼吸などに応じて顔の角度を上下に変え、鼻の洞窟もそれに追従して角度が変わる
- ゲーム開始時は洞窟が水平になるよう顔・洞窟の角度を調整する
- 主人公は鼻の入り口より少し進んだ位置からスタート

### 5.2 主人公の操作
入力: 左右移動 / ジャンプ / うつ伏せ / 鼻毛掴まり（の4系統。スマホでは最適な仮想UI）

| 状態 | 説明 |
|---|---|
| 直立 | y軸に平行に体を保つ。移動が速く、鼻への刺激が少ない |
| うつ伏せ | 地面に沿う姿勢。移動が遅く、動くと鼻への刺激が大きい |

- ジャンプ: 両状態で性能差なし。天井にぎりぎりぶつからない程度のジャンプ力。うつ伏せ中にジャンプすると直立に戻る
- 鼻毛掴まり: 鼻毛が近くにあるとき掴まりボタンで鼻毛の**先端**に追従して掴まる
  - 掴まり中は移動不可
  - 解除条件: もう一度同じボタンを押す / 鼻毛が抜ける
- 大くしゃみ発生時: 主人公に鼻の入り口方向へ大きな速度が加算される

### 5.3 くすぐりメーター（ヤギ）
- 常に少しずつ減少する
- 増加要因:
  - 主人公が地面を歩く（うつ伏せ移動は増加量が大きい）
  - 主人公が地面や天井にぶつかる
  - 鼻毛が抜ける → **一気に満タン**
- 満タンになると「くしゃみシーケンス」が始まる:
  1. 短い連続呼吸を3回（sneeze1.wav, sneeze2.wav, sneeze3.wav）
     - 呼吸のたびにヤギの顔角度を上方向に細かく段階的に上げる
  2. 大きなくしゃみ1回（sneeze_last.wav）
     - 顔角度を急速に元の水平へ戻す
     - 主人公に入り口方向の大きな速度を加算
  3. くしゃみの瞬間にメーターを 0 にし、以降は通常運用を再開

### 5.4 鼻毛の耐久
- 各鼻毛に耐久メーターがある
- 主人公が掴まっている間、静止していても少しずつ減少。主人公の速度に比例してさらに減少
- 耐久 0 で鼻毛が抜ける:
  - 掴まり状態を強制解除
  - くすぐりメーターが一気に満タン
  - 一定時間後、同じ場所に再生する

### 5.5 ゴール（ぬいぐるみ）
- 鼻の奥（耳のあたり）にぬいぐるみが挟まっている
- 近くで「鼻毛掴まりボタン」を押すと引き抜き開始:
  - 引き抜き中はくすぐりメーターが上昇
  - ぬいぐるみの耐久メーターが少しずつ減少
- ぬいぐるみ耐久 0 → **ゲームクリア確定** → クリア演出へ

### 5.6 ゲームオーバー条件
- 鼻の入り口より外へ出た瞬間にゲームオーバー確定 → ゲームオーバー演出へ

## 6. 終了演出

### 6.1 ゲームオーバー演出
1. 主人公が鼻の入り口から伸びる鼻水になって外へ飛び出す
2. ヤギが口を開け、鼻水ごと主人公を吸い込んで口の中へ
3. ゲームオーバーUIを表示（リトライ / タイトルへ戻る）

### 6.2 クリア演出
1. 鼻の中（主人公含む）が見えなくなると同時に、ヤギが通常より少し大きなくしゃみを始める
   - この間、耳のちょうど下あたりにいる主人公が常に画面中心に来るようにカメラを動かし続ける
2. くしゃみと同時に、ぬいぐるみを抱いた主人公がヤギの口から飛び出す
3. 口から出て約1秒後にゲームクリアとし、UIを表示（ゲームオーバーと同じ選択肢）
